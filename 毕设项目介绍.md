# 基于Flutter+RAG+多模态理解的小众手作艺术品定制应用设计与实现

## 一、项目简介

### 1.1 项目背景
随着消费升级和个性化需求的增长，手作艺术品市场逐渐兴起。传统手作艺术品交易存在信息不对称、定制流程不透明、信任机制缺失等问题。本项目旨在构建一个集AI推荐、多模态理解、区块链溯源于一体的手作艺术品定制平台。

### 1.2 项目目标
- 为用户提供个性化手作艺术品定制服务
- 为匠人提供作品展示和接单平台
- 为管理者提供数据化运营工具
- 通过AI技术提升定制体验和匹配效率

### 1.3 项目规模
| 模块 | 文件数 | 主要技术 |
|------|--------|----------|
| Flutter App | 109个Dart文件 | Riverpod + Dio + GoRouter |
| Spring Boot | 13 Controller + 13 Service | MyBatis-Plus + Redis + Flowable |
| Vue3 Admin | 20+页面 | Element Plus + ECharts |
| AI Service | 10+模块 | RAG + 多模态理解 + Milvus |

### 1.4 代码质量
| 指标 | 数量 |
|------|------|
| 单元测试 | 17个（UserService 7个 + ProductService 6个 + OrderService 4个） |
| 编译错误 | 0个（全部模块编译通过） |
| 安全修复 | JWT鉴权 + CurrentUserUtil统一身份提取 |
| 生产配置 | application-prod.yml（环境变量配置） |

## 四、架构亮点

### 4.1 三层架构
```
Controller → Service → Repository
   ↓           ↓          ↓
 路由分发    业务逻辑    数据访问
 参数校验    事务管理    SQL执行
```

### 4.2 安全设计
- **JWT鉴权**：无状态Token认证，24小时过期
- **CurrentUserUtil**：统一用户身份提取，防止越权攻击
- **密码加密**：SHA-256加盐哈希存储
- **Mock模式**：支付/区块链可切换，开发环境安全

### 4.3 缓存策略
- **Redis缓存**：商品列表、详情页，10分钟TTL
- **Cacheable注解**：声明式缓存，自动失效
- **缓存穿透防护**：空值缓存

### 4.4 异步处理
- **RabbitMQ**：订单状态变更通知
- **AI服务异步调用**：不阻塞主流程
- **WebSocket**：实时消息推送

## 五、项目难点与解决方案

### 5.1 多模态理解
**难点**：如何让AI理解用户的手作需求描述和参考图片？
**方案**：
- RAG检索增强：向量检索相似作品案例
- 多模态融合：文本+图片特征联合分析
- Milvus向量库：高效相似度检索

### 5.2 实时对话
**难点**：用户与AI的实时交互体验
**方案**：
- WebSocket双向通信
- 消息队列异步处理
- 流式响应返回

### 5.3 工作流引擎
**难点**：订单状态流转的灵活性
**方案**：
- Flowable BPMN工作流
- 可配置状态机
- 事件驱动架构

### 5.4 区块链溯源
**难点**：手作艺术品的真伪验证
**方案**：
- 存证记录上链
- 哈希值校验
- Mock模式可切换

## 七、安全修复

### 7.1 ChatController安全漏洞
- **问题**：userId从请求参数/Body获取，可被伪造
- **修复**：所有接口从JWT Token提取userId，使用`CurrentUserUtil`统一处理
- **影响**：6个Controller（Chat、Favorite、User、Artisan、History、Blockchain、Order、Payment）

### 7.2 CurrentUserUtil工具类
- **功能**：封装JWT Token解析，提供`extractUserId(request)`方法
- **位置**：`server/src/main/java/com/jiangwu/utils/CurrentUserUtil.java`
- **使用**：所有需要用户身份的Controller统一调用

### 7.3 生产环境配置
- **文件**：`application-prod.yml`
- **特点**：使用环境变量替代硬编码，关闭Mock模式，优化连接池
